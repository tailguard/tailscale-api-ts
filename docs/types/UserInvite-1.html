A user invite is an active invitation that lets a user join a tailnet with a preassigned user role.

Each user invite has a unique ID that is used to identify the invite in API calls. You can find all user invite IDs for a particular tailnet by listing user invites.

Type declaration

  • Optionalemail?: string

    The email to which the invite was sent. If empty, the invite was not emailed to anyone, but the inviteUrl can be shared manually.

  • id: string

    The unique identifier for the invite. Supply this value wherever userInviteId is indicated in the endpoint.

  • OptionalinviteUrl?: string

    Included when email is not part of the tailnet's domain, or when email is empty. It is the link to accept the invite.

    When included, anyone with this link can accept the invite. It is not restricted to the person to which the invite was emailed.

    When email is part of the tailnet's domain (has the same @domain.com suffix as the tailnet), the user can join the tailnet automatically by logging in with their domain email at https://login.tailscale.com/start. They'll be assigned the specified role upon signing in for the first time.

  • inviterId: number

    The ID of the user who created the invite.

  • OptionallastEmailSentAt?: string

    The last time the invite was attempted to be sent to Email. Only ever set if email is not empty.

  • role: UserInvite.role

    The tailnet user role to assign to the invited user upon accepting the invite.

  • tailnetId: number

    The ID of the tailnet to which the user was invited.
